# inputs/

사용자가 팀에게 주는 모든 자료는 이 디렉토리에 둡니다.
은광은 작업 시작 시 자동으로 이 폴더를 스캔하고 자료를 팀원에게 분배합니다.

## 하위 디렉토리 규칙

| 디렉토리 | 용도 | 1차 분석 담당 |
|----------|------|--------------|
| `design/` | 디자인 시안, 와이어프레임, 브랜드 가이드, 스크린샷 | 현식 |
| `requirements/` | 요구사항 문서, 스펙, 사용자 스토리, ADR | 민혁 + 프니엘 |
| `data/` | CSV / JSON / 엑셀 / 샘플 데이터 | 프니엘 + 창섭 |
| `code/` | 기존 코드, 참고 코드, 라이브러리 zip 등 | 창섭 + 민혁 |
| `reference/` | 참고 URL 모음, 경쟁 서비스 캡처, 외부 문서 | 프니엘 + 현식 |

위 분류에 안 맞으면 자유롭게 새 디렉토리를 만들거나 루트(`inputs/`)에 그냥 둬도 됩니다.
은광이 파일명/확장자로 자동 분류합니다.

## 명명 컨벤션 (선택)

명확성을 위해 다음 패턴을 권장합니다 — 강제는 아닙니다.

- 디자인: `{화면명}-{버전}.png` 예: `home-v2.png`, `signup-flow.pdf`
- 요구사항: `{topic}-spec.md` 예: `auth-spec.md`, `payment-requirements.md`
- 데이터: `{entity}-{date}.csv` 예: `users-2025-11.csv`

## 사용 방법

1. 자료를 적절한 하위 디렉토리에 둔다.
2. 현재 Codex 대화에서 은광에게 한 줄로 요청한다.
   ```
   inputs 보고 랜딩 페이지 만들어줘
   ```
   또는
   ```
   inputs/design/ 시안대로 회원가입 페이지 만들어줘
   ```
3. 은광이 자동으로:
   - `inputs/` 스캔
   - 파일별 요약
   - 규모 판정 + Triple Crown 워크플로우 선언
   - 자료 경로를 팀원 또는 직접 처리 흐름에 절대 경로로 전달
   - Phase 1~5 실행

## 큰 파일 / 민감 데이터

- 1GB 이상 파일은 git에 커밋하지 않습니다 (`.gitignore` 등록 권장).
- 개인정보가 포함된 자료는 마스킹 후 두거나, 별도 디렉토리(`inputs/private/`)를 만들어 `.gitignore`에 추가합니다.
